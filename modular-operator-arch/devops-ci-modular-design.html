Recommendation

Option A (Standard Konflux Nudging) for the initial rollout.

Reasons:
  • Reuses proven infrastructure already in production
  • Zero custom development needed
  • Full git audit trail at every level
  • The latency increase (~15-20 min additional) is acceptable for CI
  • The build chain is NOT on the critical path for developer iteration

Option B can be evaluated later as an optimization once modular architecture is stable.
